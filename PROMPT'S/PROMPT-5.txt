[ESPECIFICAÇÃO TÉCNICA]
Implemente o módulo de Quiz (múltipla escolha), o Desafio Sequencial e o jogo de
Completar Lacunas. Respeite obrigatoriamente:

Stack já definida: React Native + Expo + TypeScript strict + NativeWind +
AsyncStorage + Expo Router.

Requisitos a implementar neste prompt:
- RF01: Quizzes de múltipla escolha; Desafios sequenciais; Jogos de completar
  lacunas em sentenças ou fórmulas
- RF02: Sequência de temas definida pelo sistema, sem escolha livre
- RF03: Perguntas cobrindo definições/conceitos, fórmulas/cálculos e aplicações
  práticas
- RF04: Entre 3 e 5 temas distintos
- RF05: Feedback imediato com explicação após cada resposta
- RF06: Registro de acertos e tempo gasto por sessão
- RF07: Questões agrupadas por dificuldade
- RNF02: Botão de reinício a qualquer momento
- RNF04: Dicas/explicações extras para respostas incorretas

A experiência do usuário deve seguir o fluxo:
Quiz → escolha do módulo → escolha da dificuldade ou todas → sessão com perguntas
sequenciais → resultado final.

As perguntas são conteúdo interno da sessão. Elas não devem aparecer como itens
principais de navegação ou cards individuais na tela inicial do quiz.

Schema conceitual do banco de questões — os JSONs devem representar uma coleção
de módulos de estudo, e não uma lista plana de perguntas. Cada módulo deve possuir
seus próprios metadados e sua coleção de questões:
{
  "modulos": [
    {
      "id": "magnetismo",
      "titulo": "Magnetismo",
      "descricao": "Conceitos fundamentais de magnetismo",
      "questoes": [
        {
          "id": "mag_q_001",
          "tipo": "<multipla_escolha | calculo>",
          "enunciado": "<pergunta completa, com dados se for cálculo>",
          "alternativas": [
            {"letra": "A", "texto": "..."},
            {"letra": "B", "texto": "..."},
            {"letra": "C", "texto": "..."},
            {"letra": "D", "texto": "..."}
          ],
          "resposta_correta": "",
          "explicacao": "<resolução comentada, mostrando o raciocínio>",
          "dificuldade": "<facil | medio | dificil>",
          "fonte": "..."
        }
      ]
    }
  ]
}

Schema do banco de lacunas — consumir data/lacunas.json:
{
  "lacunas": [
    {
      "id": "lac_mag_001",
      "modulo_id": "magnetismo",
      "frase": "A unidade de medida da densidade de campo magnético no SI é _______.",
      "resposta": "tesla",
      "respostas_aceitas": ["tesla", "T"],
      "explicacao": "Explicação didática exibida após a resposta.",
      "dificuldade": "<facil | medio | dificil>",
      "fonte": "..."
    }
  ]
}

Cada lacuna deve possuir modulo_id para indicar a qual módulo pertence.
Não inferir o módulo pelo prefixo do id.

Entregáveis obrigatórios:
1. app/jogos/quiz/index.tsx — tela de preparação da sessão de quiz.
   Deve apresentar os módulos de estudo disponíveis e permitir escolher a

   individuais.
2. app/jogos/quiz/[modulo].tsx — sessão de quiz com perguntas sequenciais
   carregadas a partir do módulo selecionado e filtradas pela dificuldade

3. app/jogos/desafio/[modulo].tsx — desafio sequencial usando as questões
   do módulo selecionado, com filtro de dificuldade quando informado, uma questão

4. app/jogos/lacunas/[modulo].tsx — sessão de completar lacunas carregando
   itens de data/lacunas.json pelo modulo_id selecionado e filtrando por
   dificuldade quando informado. Deve exibir uma frase por vez, campo de input,
   validação com respostas_aceitas, feedback imediato e resultado final.
   Use Header.tsx e BotaoReiniciar.tsx já existentes
5. components/AlternativaButton.tsx — botão de alternativa com estados: neutro,

6. components/EnunciadoCard.tsx — card de enunciado reutilizável nos três modos.
